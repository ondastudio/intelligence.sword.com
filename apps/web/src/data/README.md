# Content & data

Editable content lives here, kept separate from code so **non-developers can
change text, links, and settings without touching components.**

Use plain JSON or Markdown files (e.g. `site.json` for the site name, nav links,
and footer; one file per page or section for body content). Components and pages
import from this folder and render the data — so editing a value here updates the
site, with no code changes.

Structure is added as the design and pages are built out.
